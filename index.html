今天练腿。

深蹲前的热身、松解、拉伸和激活确实对蹲起的表现有加成，感觉蹲的更稳了，脚掌的蹬地感也很强，另外没有出现之前下蹲时侯右侧膝盖的微微弹响。总之很值得每次练腿之前做一下。主要参考的是凯圣王的【收藏向】功能热身——给你的深蹲加个BUFF

具体的项目包括

  1. 热身：自行车低档位骑5分钟，当全身到达一个微微热的状态。具体的档位看个人，主要是能感到一定的阻力就行，毕竟后面深蹲有的是要用力的地方，不要在这里浪费糖原。这一步主要是让身体进入一个运动的状态，所谓的分解代谢。
  2. 松解：松解的话主要是放松紧张的相关肌群。因为是练腿的缘故，我主要是用泡沫轴按压小腿后侧以及腘绳肌（即大腿后侧），避免太紧张从而借力。组数大概是每侧每个部位滚动20次，感觉松软了即可。
  3. 拉伸：练腿的拉伸我选择了最伟大的拉伸、弓箭步拉伸、以及青蛙趴。
    其中，最伟大的拉伸是一个动态拉伸，可以保持身体热度的同时，拉伸我们的后侧链以及核心。弓箭步拉伸则是拉伸我们的喀腰肌，因为我们深蹲包括一个屈髋的动作，这有助于降低我们髋部弹响的概率。最后一个青蛙趴也是一个针对髋部的静态拉伸。组数的话，最伟大的拉伸两侧算一组做10组，然后弓步单侧10次，青蛙趴30秒。

  1. 活动关节：深蹲是一个多关节复合性动作，包括踝关节、膝关节和髋关节。其中，踝关节通过较大角度的单侧旋转完成，也就是我们中学体育老师教过的“转脚尖”。然后膝关节的活动我是通过器械完成的，但是也可以通过将屈膝90度靠墙静坐，保持15-30秒。最后髋关节则是通过早安式体前屈。

  2. 激活：激活肌肉的目的是提前让目标肌肉进入状态。如果只拉伸但是不激活，那么运动的发力感则会不够。我的激活动作一组是无负重深蹲，然后一组无负重保加利亚深蹲。

最后附上今天的训练记录。

通信链路的方向

2023-09-05
blog

两个通信设备之间的连接称为通信链路。如果链路是通过多路复用技术构建的，则也称为虚拟信道。通信链路和虚拟信道并不一定相同，这与通信链路的方向性有关。一般来说，通信链路有三种方向性：

  • 全双工：这意味着链路是双向的，双方可以同时向对方发送信息。然而，虚拟信道通常是单向的。要实现全双工传输，链路必须使用两个虚拟信道，每个信道负责一个方向的传输。例如，宽带网络中的 Cable Modem 和 ADSL 网络有两个信道，一个用于上行链路，一个用于下行链路，以形成双向全双工传输。但是，两个信道的速率不一定相同。

  • 半双工： 半双工链路是指通信设备允许双向通信，但不能同时传输数据，必须交替进行。半双工链路只使用一个虚拟信道，通信两端轮流使用该信道：获得访问权的通信端可以传输数据，否则只能准备接收数据。虽然半双工链路的信道数量较少，但管理起来却更加复杂。一般来说，访问权限管理使用令牌传输，获得令牌后即可传输数据。

  • 单工：这意味着通信链路只允许单向传输数据。单工链路只需要一个虚拟通道。链路的一侧负责传输数据，另一侧只能接收数据，一般用于广播信息。

用HEXO和Github建立个人网站

2023-09-03
blog

我的个人网站 https://xiaowangya.github.io/

准备

环境配置

  • Node.js 安装

    根据自己的电脑平台从该下载链接安装。我个人是windows平台，选择的是长期维护版的“64bit.msi”文件。

    双击运行安装文件，安装过程全程下一步，记得勾选 自动加入Path 选项。

    因为Npm是Node.js的包管理工具(package manager)，所以在安装Node.js的时候会自动安装npm.

    检查是否安装成功

      npm -V #检查npm安装
  node -v #检查node.js
  path #检查环境变量

  • Git安装

    见此教程

框架和主题安装

  • HEXO框架安装

    随后打开CMD, 在命令行中输入

      npm install -g hexo-cli

    其他平台（如 MAC）参见link

    安装hexo插件

      npm install hexo-deployer-git --save

    在下载中意的主题，并在对应的github网站处下载。本方案选择的是AYER主题。

修改个人信息

选择网站主题

首先，创建一个本地用于存储网站的文件夹。然后在该文件夹中，执行

git init
hexo init dir_name #dir_name是你定义的文件夹名字
cd dir_name
npm install

在HEXO中选好主题并安装到themes文件夹下

cd themes
git clone https://github.com/your_theme.git #前面选好的主题对应的git仓库

更新相关信息

  1. github 修改

在GitHub上一个新的仓库，命名为yourwebname.github.io。
然后在unwatch选项上方的setting中，左侧栏选择code and automation/page，修改对应的url, source以及branch信息

  1. 本地修改

    用编辑器打开文件夹中的_config.yml文件，并修改一下关键信息

     url: https://username.github.io/project
 
 deploy:
     type: 'git'
     repo: https://github.com/XiaoWangya/xiaowangya.github.io
     branch: master
 theme: hexo-theme-next

  2. 其他信息

    还是在上面这个文件中，修改对应的姓名等信息

发布你的第一个文章

命令行输入

hexo new "blog"

创建新博客，然后在./source/_posts/blog.md处可见新的.md文件。具体的语法见此处

编辑完成之后，在命令行输入

hexo g && hexo d

稍等片刻网站即会完成更新。

其他

想要删除以前的文档，可以在本地删除对应文件之后，执行

hexo clean
hexo g && hexo d

即可
